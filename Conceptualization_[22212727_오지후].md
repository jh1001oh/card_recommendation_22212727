1. Business purpose
   
1.1 Project background
   
![](https://github.com/jh1001oh/conceptualization--22212727-/blob/main/%EC%8B%A0%EC%9A%A9%EC%B9%B4%EB%93%9C%20%EC%9D%B4%EC%9A%A9%EC%8B%A4%EC%A0%81.png?raw=true)

 현대사회에서 신용카드 이용 실적은 꾸준히 증가하고 있다. 소비자들은 이제 결제 수단을 넘어 다양한 혜택과 맞춤형 서비스까지 고려하며 카드를 선택하고 있다.
특히 온라인 쇼핑, 배달 서비스, ott가 일상화되며 카드 사용액은 생활 전반에 걸쳐 확대되고 있다. 이에 발맞춰 카드사들은 고객의 소비 데이터를 분석해 맞춤화된 혜택을 제공하려는 경쟁이 심화되고 있다.
하지만 수많은 카드가 경쟁적으로 나올수록 소비자는 자신의 소비 성향에 핏하는 카드를 찾는 게 어려워지고 있다. 나의 경우에도 나에게 핏하는 카드를 찾기 위해 며칠간 비교했지만 결국엔 찾지 못 한 경험이 있다. 
이처럼 카드사별로 제고하는 할인이나 적립 혜택이 각기 다르기에 사용자가 직접 비교하기엔 많은 시간과 노력이 필요하다. 예를 들어 한 카드는 외식이나 배달에 큰 할인을 제공하지만 온라인 쇼핑에는 혜택이 미미할 수도 있고 다른 카드는 주유시 적립률이 높지만 ott에는 혜택이 없을 수가 있다.
 이러한 어려움을 해결하고자 사용자의 월 평균 지출 패턴을 기반으로 맞춤형 신용카드를 추천하는 어플리케이션을 개발하는 것을 목표로 한다. 사용자가 주요한 카테고리 별로 자신의 월 평균 지출액을 입력하면 다양한 카드의 혜택을 자동으로 분석하여 사용자가 가장 높은 할인 및 정립을 받을 수 있는 카드를 추천해서 제공한다.
단순한 카드 정보 제공에서 나아가 사용자가 실질적으로 경제적인 이익을 극대화할 수 있도록 맞춤형 해결방안을 제공하는 것이다. 또한 사용자가 본인의 지출액을 객관적으로 파악하게 함으로써 합리적인 소비 습관 형성을 도울 수 있다.

1.2 Goal

신용카드를 추천해주는 어플리케이션 개발
사용자가 카테고리별로 월평균 지출액을 입력하면 이에 맞는 신용카드 혜택 계산
가장 높은 할인과 적립을 받을 수 있도록 추천

1.3 Target market

자신의 소비 습관에 맞는 카드를 추천받고 싶은 소비자

2. System context diagram

![](https://github.com/jh1001oh/conceptualization--22212727-/blob/main/System%20Diagram%20%EC%BA%A1%EC%B3%90%EB%B3%B8.png?raw=true)

User 사용자
: 신용카드 추천 앱을 실제로 사용하는 소비자, 소비 카테고리별 월평균 지출액을 입력한다.

Core 핵심 기능
: 카드를 추천하는 알고리즘으로 카드의 혜택과 피킹률을 계산한다.

Data
: 카드와 사용자 소비 데이터를 저장한다.

System
: Core와 Data로 구성된 시스템

3. Use case list

1) 회원가입

    Actor: User
   Description: 앱에 새로 가입한다. 이메일, 비밀번호, 이름 등을 입력하여 계정을 생성한다.
2) 로그인 / 로그아웃

    Actor: User
   Description: 등록된 사용자가 입력한 이메일과 비밀번호로 로그인하거나 로그아웃을 한다.
3) 지출 정보 입력

    Actor: User
   Description: 사용자가 카테고리별로 월평균 지출액을 입력한다.
4) 카드 추천 결과 조회

    Actor: User
   Description: System이 분석해서 추천한 카드 목록과 예상 할인 및 적립 금액을 조회한다.
5) 카드 상세 정보 확인

    Actor: User
   Description: 추천된 카드의 혜택 조건과 연회비 등을 확인한다.
6) 지출 내역 수정

    Actor: User
    Description: 이전에 입력한 지출 정보를 수정한다.
7) 카드 DB 관리

   Actor: 관리자
   Description: 새로운 카드를 추가하거나 기존 정보를 수정 또는 삭제를 한다.

4. Concept of operation

1) 회원가입
Purpose
:앱을 사용하기 위해 계정을 만든다.
Approach
:사용자가 이메일, 비밀번호, 이름을 입력하고 회원가입을ㅇ 누르면 서버는 이메일 중복 여부를 확인한 후 계정을 생성한다. 가입 성공 시 로그인 화면으로 이동하고 실패 시 오류 메세지를 출력한다.
Dynamics
:계정이 없는 경우
Goals
:회원가입 기능을 구현한다.

2) 로그인 / 로그아웃
Purpose
:앱을 사용하기 위해 등록된 사용자인지 확인한다.
Approach
:사용자가 이메일과 비밀번호를 입력하고 로그인을 누르면 서버에서 회원 정보를 조회하여 로그인 성공/실패 여부를 확인한다. 로그아웃 시 로그인 화면으로 이동한다.
Dynamics
:앱 실행 시 로그인하는 경우 / 앱 종료 시 로그아웃하는 경우
Goals
:로그인 / 로그아웃 기능을 구현한다.

3) 지출 정보 입력
Purpose
:사용자의 소비 데이터를 수집한다.
Approach
:사용자의 월평균 지출액을 입력한다. 카테고리는 식비, 온라인 쇼핑, 배달, OTT, 주유 등으로 나눈다.
Dynamics
:처음 앱을 사용할 경우
Goals
:카드 추천을 위한 사용자 데이터를 수집한다.

4) 카드 추천 결과 조회
Purpose
:사용자의 지출 패턴에 맞는 신용카드를 추천한다.
Approach
:서버가 사용자의 데이터와 카드 혜택 데이터를 비교하여 사용자가 받을 수 있는 할인, 적립혜택이 가장 큰 카드 순으로 정렬한 추천 목록을 생성한다. 결과에는 카드 이름, 연회비, 예상되는 월 혜택 금액을 표시한다.
Dynamics
:지출 정보 입력 후 또는 추천 결과 다시 보기 시
Goals
:사용자 맞춤형 카드를 추천한다.

5) 카드 상세 정보 확인
Purpose
:추천된 카드의 상세 혜택 정보를 확인한다.
Approach
:해당 카드의 할인율, 적립율, 전월 실적 조건, 연회비, 카드사 등을 보여준다.
Dynamics
:추천 카드 결과에서 한 카드를 선택한 경우
Goals
:카드의 상세 혜택을 보여주면서 사용자가 전체적인 정보를 알 수 있게 한다.

6) 지출 내역 수정
Purpose
:바뀐 지출을 반영하여 결과에 반영한다.
Approach
:지출 입력칸에 들어가서 기존의 지출액을 수정 또는 추가한다. 수정이 완료되면 자동으로 추천 결과가 갱신된다.
Dynamics
:처음 입력한 지출 내역을 변경하고 싶은 경우
Goals
:바뀐 지출 내역을 반영한다.

7) 카드 DB 관리
Purpose
:항상 카드 데이터를 최신으로 유지한다.
Approach
:관리자가 카드 정보를 변경한다. 변경된 내용은 즉시 반영해서 사용자의 추천에 적용이 되게 한다.
Dynamics
:카드의 혜택이 변경되거나 새로운 카드가 출시된 경우
Goals
:정확한 추천을 제공한다.

5. Problem statement

1) 카드 혜택 데이터 수집
   :초기에 카드 데이터를 수집할 때 수작업이 필요하다.
2) 카드 혜택 데이터 최신화
   :수시로 바뀌는 혜택을 지속적으로 최신의 상태로 유지하는 것에는 큰 노력이 필요하다.
3) 사용자 보안
   :민감한 데이터이기 때문에 인증 등 보안 문제에 큰 관심을 쏟아야 한다. 또한 사용자들이 민감 데이터이기에 작성하는데 진입장벽이 있을 수가 있다.

6. Glossary

- 할인
  :결제 금액 중 일정 금액을 카드사에서 차감해주는 금액
- 적립
  :결제 금액에 비례하여 포인트나 마일리지를 쌓아주는 혜택
- 전월 실적
  :카드 혜택을 받기 위해 카드로 승인된 금액 중 실적으로 인정되는 항목만 합산한 금액(전달에 해당 카드로 사용해야 하는 최소 결제 금액)
- 연회비
  :카드를 사용하는 대가로 일 년에 한 번씩 납부해야 하는 금액
- 지출 카테고리
  :사용자가 소비하는 항목을 분류해서 구분
- 추천 알고리즘
  :사용자의 지출 데이터를 분석해서 카드의 혜택 조건을 비교하여 예상 헤택 금액이 가장 높은 카드를 찾아 제시하는 로
- OTT
  :인터넷을 통해 각종 미디어 콘텐츠를 제공하는 스트리밍 서비스
- 피킹률
  :카드로 결제한 금액 대 비 실제로 돌려 받은 혜택의 비율
