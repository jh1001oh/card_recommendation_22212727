# 1. Introduction
   
## 1) Executive Summary
   
 현대 사회에서 신용카드는 단순한 결제 수단을 넘어 다양한 혜택과 서비스를 제공하는 핵심 금융 도구가 되었다. 그러나 다양하게 유통되고 있는 카드 중 자신의 소비 패턴에 맞는 카드를 직접 비교하고 선택하는 것은 일반 소비자에게 매우 어렵고 시간이 많이 걸리는 일이다. 지난 개념화 문서에서 정의한 요구사항을 바탕으로 하여 사용자가 카테고리별 월평균 지출액을 입력하면 최적의 신용카드를 자동으로 추천해주는 앱 'Card Fit'의 분석 단계를 기술한다.
 시스템이 무엇을 하는가(what)에 초점을 맞추어 Use case analysis, Domain analysis, User Interface prototype을 다룬다. 각 장에서는 UML을 활용한 분석 결과와 그 의도를 함께 기술하였다.

## 2) Business Goals
   
 Card Fit은 사용자가 자신의 소비 습관에 맞는 신용카드를 쉽고 빠르게 찾을 수 있도록 지원하는 것을 목적으로 한다. 외식, 쇼핑, 통신, OTT, 주유, 편의점 등 카테고리별 지출액을 입력하면 카드별 혜택을 자동으로 계산하여 예상 할인·적립 금액이 높은 순서대로 추천 결과를 제공한다. 단순한 정보 제공을 넘어 사용자가 실질적인 경제적 이익을 극대화할 수 있도록 맞춤형 솔루션을 제시하는 것이 핵심 목표이다.

## 3) Technical Goals
   
 Card Fit은 모바일 앱 형태로 제공되며 User와 Administrator가 서버의 카드 혜택 DB와 통신하는 구조로 설계한다. 카드 혜택 계산 알고리즘은 전월 실적 조건, 카테고리별 할인·적립률, 월 한도 등의 조건들을 정확하게 처리해야 하며 추천 결과는 입력 완료 후 3초 이내로 화면에 출력되어야 한다. 사용자 데이터는 암호화하여 저장하고  Administrator가 카드 정보를 편리하게 관리할 수 있는 인터페이스도 함께 제공한다.

# 3. Domain analysis

**1) User**

사용자의 기본 정보(이메일, 비밀번호, 닉네임)를 저장하는 클래스이다.

**2) Administrator**

관리자 권한을 가진 클래스이다. User 클래스를 상속한다.

**3) SpendingData**

사용자의 카테고리별 월평균 지출액을 저장하는 클래스이다. 카테고리는 외식, 쇼핑, 통신, OTT, 주유, 편의점 등으로 구성되고 각 카테고리의 금액 속성을 가진다. RecommendEngine 클래스가 이 데이터를 참조한다.

**4) CardInfo**

신용카드 정보를 나타내는 클래스이다. 이 클래스는 카드명, 발급사, 연회비, 카테고리별 할인율과 적립율, 전월 실적 조건, 월 최대 혜택의 속성을 가진다. Administrator가 CardInfo를 관리한다.

**5) BenefitCondition**

CardInfo와 연관되는 클래스이다. 위에서 정리한 혜택 별로 세부조건을 저장한다. 한 카드는 다수의 BenefitCondition을 가질 수 있다.

**6) RecommendEngine**

사용자의 SpendingData와 전체 Cardinfo를 비교해서 예상되는 혜택 금액을 계산하고 카드의 순위를 매기는 클래스이다. 가장 중요한 핵심 알고리즘으로 계산된 결과를 RecommendResult로 반환한다.

**7) RecommendResult**

RecommendEngine의 결과를 저장하는 클래스이다. 사용자가 추천 결과 화면에서 직접 조회할 수 있는 데이터이다.

**8) CompareResult**

사용자가 선택한 2~3개의 카드를 비교한 결과를 저장하는 클래스이다. 사용자가 직접 선택한 카드들의 각 CardInfo와 RecommendResult를 함께 참고해서 화면을 구성한다.

**9) CardDB**

카드 혜택 DB 역할을 하는 클래스이다. 모든 CardInfod와 BenefitCondition 데이터를 저장하고 관리한다. RecommendEngine과 Administrator가 이 클래스를 통해서 정보에 접근할 수 있다.

**10) AuthService**

회원가입, 로그인, 로그아웃 등의 기능을 처리하는 클래스이다. 이 클래스로 User인지 Administrator인지의 자격도 검증한다.
